## ✨ Core Features

### 🧠 Productivity Engine

ProTrack's task management system is built around a structured **Kanban-style workflow** that mirrors professional sprint methodologies — adapted for personal use.

- **Smart Task Creation** — Add tasks with title, description, due date, priority, tags, and subtasks in a single unified modal
- **Priority Intelligence** — Three-tier priority system (`High` / `Medium` / `Low`) with visual differentiation and urgency signaling
- **Kanban Workflow** — Drag-and-drop-ready status movement across `Pending → In Progress → Completed`
- **Live Overdue Detection** — Automatic visual flagging of overdue tasks with deadline countdowns
- **Advanced Filtering** — Combine status, priority, date range, and keyword search to instantly surface the right tasks
- **Subtask Support** — Break complex tasks into granular steps and track completion at each level
- **Recurring Routines** — Schedule repeating tasks daily, weekly, or custom intervals for consistent habit tracking

---

### 💪 Fitness Intelligence

Move beyond simple step counters. ProTrack's fitness module creates a **personalized health narrative** built from daily logs, goal comparisons, and streak discipline.

- **Activity Logging** — Record steps taken, workout duration (minutes), activity type (run, strength, yoga, etc.), and session notes
- **Adaptive Goal Engine** — Set unique daily targets for steps and active minutes; goals update performance metrics in real-time
- **Streak Analytics** — Monitor current streaks and longest streaks to build accountability and sustain momentum
- **Trend Indicators** — See at a glance whether today's performance is higher or lower than yesterday, last week, and last month
- **Historical Timelines** — Browse your entire activity log in reverse chronological order for a complete wellness picture

---

### 📊 Analytics & Reporting

ProTrack transforms raw data into **clear, actionable insights** through a powerful reporting layer.

- **Weekly & Monthly Snapshots** — Aggregated summaries with comparison to prior periods
- **Task Completion Rates** — Percentage-based tracking of high/medium/low priority completion
- **Fitness Goal Hit Rate** — Daily success rate across step and workout targets
- **Activity Heatmap** — A GitHub-style contribution heatmap visualizing consistency at a glance
- **Chart.js Visualizations** — Beautiful, animated bar, line, and doughnut charts render complex data simply
- **Momentum Analysis** — Identify your best days, peak performance windows, and areas needing attention

### Architectural Decisions

#### Why Vanilla JS?

ProTrack deliberately avoids frameworks like React or Vue to:
1. **Eliminate load overhead** — No hydration, no virtual DOM diffing, no bundle parsing
2. **Ship instantly** — Drag files to any CDN and it works. No `npm install`, no `npm run build`
3. **Demonstrate mastery** — Framework abstractions can hide complexity. Vanilla JS makes it explicit

#### Why localStorage?

- **Zero latency** — Reads and writes are synchronous and instant
- **Zero cost** — No database, no server, no monthly bill
- **Privacy first** — Your data never leaves your device
- **Offline resilient** — Works with no internet connection whatsoever

#### CSS Custom Properties Architecture

All design tokens (colors, spacing, radius, shadows, transitions) are defined as CSS custom properties on `:root`, enabling:
- Instant theme switching with a single class toggle
- Consistent visual language across all components
- Trivial customization for forks

## 🔐 Security & Privacy

ProTrack is a **local-first, privacy-respecting application**:

- 🔒 **No data transmitted** — All user data lives exclusively in your browser's localStorage
- 🚫 **No accounts required** — The demo login is purely client-side; no credentials are sent anywhere
- 📦 **Data portability** — Full export/import support lets you own and migrate your data
- 🛡️ **No tracking** — No analytics scripts, no telemetry, no third-party data collection
- 💾 **Backup regularly** — Since data is device-specific, use the Export feature as a backup habit

> ⚠️ **Note:** Clearing browser data or localStorage will erase stored tasks and fitness logs. Export your data periodically using the in-app backup feature.

## 🚀 Deployment

ProTrack is a **pure static application** — there is no build process. Deployment is as simple as uploading three files.

### Option 1: Vercel (Recommended)

[![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/yourusername/protrack)

1. Fork this repository
2. Import into [vercel.com](https://vercel.com)
3. Click **Deploy** — done in under 30 seconds

### Option 2: Netlify

1. Drag and drop the project folder into [netlify.com/drop](https://app.netlify.com/drop)
2. Netlify auto-deploys and provides a live URL immediately

### Option 3: GitHub Pages

```bash
# In your GitHub repository settings:
# Settings → Pages → Source → Deploy from branch → main → / (root)
```

### Option 4: Local Development

```bash
# Clone the repository
git clone https://github.com/yourusername/protrack.git
cd protrack

# Open directly in browser (no server needed)
open index.html

# Or use a lightweight local server
npx serve .
# OR
python -m http.server 3000
```

### Files Required for Deployment

```
index.html   ← App entry point
style.css    ← All styling
script.js    ← All application logic
```

No `node_modules`. No `package.json`. No build artifacts. Just three files.

## 🤝 Contributing

Contributions are welcome and appreciated! ProTrack follows a straightforward contribution workflow.

### Getting Started

```bash
# 1. Fork the repository on GitHub
# 2. Clone your fork locally
git clone https://github.com/YOUR_USERNAME/protrack.git
cd protrack

# 3. Create a feature branch
git checkout -b feature/your-feature-name

# 4. Make your changes (no build step needed)
# 5. Test in your browser

# 6. Commit with a clear message
git commit -m "feat: add recurring task automation"

# 7. Push and open a Pull Request
git push origin feature/your-feature-name
```

### Contribution Guidelines

- **Code Style** — Use consistent 2-space indentation; follow existing naming conventions
- **Commit Messages** — Use [Conventional Commits](https://www.conventionalcommits.org/) (`feat:`, `fix:`, `docs:`, `refactor:`)
- **No Dependencies** — Do not introduce new npm packages or CDN dependencies without discussion
- **Test Manually** — Verify your changes work in at least two modern browsers
- **Document Changes** — Update this README if you add or change features

### Types of Contributions Welcome

- 🐛 Bug reports and fixes
- ✨ New feature implementations from the roadmap
- 🎨 UI/UX improvements and accessibility enhancements
- 📝 Documentation improvements
